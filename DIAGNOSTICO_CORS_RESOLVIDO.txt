FUNÇÃO UTILIZADA:
  getProxyUrl(firebaseUrl: string): string

  Localização: src/services/storageService.ts (Ln 539)

  O que faz:
  ├─ Extrai caminho do arquivo de URL do Firebase
  ├─ Regex: /\/o\/([^?]+)/
  ├─ Decodifica caracteres especiais
  └─ Retorna: /api/storage/download/{decodedPath}

  Exemplo:
  Input:  https://firebasestorage.googleapis.com/v0/b/gen-lang.../o/...%2Fimage.jpg?alt=media&token=xyz
  Output: /api/storage/download/generation_images/image.jpg


FALLBACK SEGURO:
  Se getProxyUrl() falhar (ex: URL em formato desconhecido):
  ├─ Captura exceção com try/catch
  ├─ Log: "[GenerationStep] Could not convert resultUrl to proxy"
  ├─ Usa URL original como fallback
  └─ Resultado: Melhor tenta que nada

ERRO 1: "Missing or insufficient permissions"
──────────────────────────────────────────────
Causa: Firebase Storage Rules não permitindo leitura

Solução:
  1. Verificar Firebase Console → Storage → Rules
  2. Garantir que: allow read: if true; (ou autenticado)
  3. Aplicar regras

Não é CORS! É Firebase Rules.


ERRO 2: "Could not convert resultUrl to proxy"
───────────────────────────────────────────────
Causa: URL em formato inesperado

Solução:
  1. Verificar console log
  2. Ver qual é o formato da URL que veio
  3. Atualizar getProxyUrl() se necessário
  4. Ou: arquivo em formato não-Firebase

──────────────────────────────────────────────────────
Causa: Arquivo não foi salvo completamente

Solução:
  1. Verificar Firestore → generation_sessions
