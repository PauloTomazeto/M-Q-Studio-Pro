================================================================================
PROBLEMA ORIGINAL
================================================================================

Arquivo: src/services/imageGenerationService.ts, linha ~67
Código:  const apiResolution = resolution === '4k' ? '4K' : (resolution === '2k' ? '2K' : '1K');

Impacto:
  • Apenas 2 resoluções funcionavam corretamente ('4k' e '2k')
  • Resoluções '3k' e '2.5k' caíam no fallback '1K'
  • Usuário selecionava 3k/2.5k, mas a imagem gerava em 1K
  • Desperdício de créditos do usuário
  • Péssima experiência do usuário

================================================================================
SOLUÇÃO IMPLEMENTADA
================================================================================

1. RESOLUTION_MAP - Mapeamento centralizado e completo

   const RESOLUTION_MAP: Record<string, string> = {
     '4k': '4K',
     '3k': '3K',
     '2.5k': '2.5K',    ← NOVO
     '2k': '2K',
     '1k': '1K'
   };

2. Função mapResolution() - Validação de entrada

   const mapResolution = (resolution: string): string => {
     const mapped = RESOLUTION_MAP[resolution.toLowerCase()];
     if (!mapped) {
       throw new Error(`Invalid resolution: ${resolution}.
                       Supported resolutions: ${...}`);
     }
     return mapped;
   };

3. Uso em startGeneration() - Integração simples

   const apiResolution = mapResolution(resolution);

Benefícios:
  ✓ Todas as 5 resoluções mapeadas
  ✓ Validação com erro descritivo
  ✓ Case-insensitive
  ✓ Código limpo e manutenível
  ✓ Type-safe
