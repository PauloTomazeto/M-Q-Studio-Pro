MAPA MENTAL: PROBLEMA → SOLUÇÃO
================================================================================

┌─────────────────────────────────────────────────────────────────┐
│                        O PROBLEMA                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ Erro Observado:                                                 │
│   Browser: "CORS Error" ou "Permission Denied"                 │
│   Gemini: ✅ Consegue acessar a mesma imagem                   │
│                                                                 │
│ Raiz Técnica:                                                   │
│   URLs públicas NÃO incluem credenciais de acesso               │
│   Firebase Storage rejeita browser sem credenciais              │
│   Gemini via proxy consegue (servidor tem credenciais)          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                    O QUE FOI DESCOBERTO                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ ✅ CORS config correto        (/cors.json)                      │
│ ✅ Firestore rules correto    (/firestore.rules)                │
│ ❌ Storage rules AUSENTE      (NÃO EXISTE - culpado!)           │
│ ❌ URLs sem credenciais        (getDownloadURL)                 │
│                                                                 │
│ Conclusão: Não é config, é arquitetura de autenticação          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                      A SOLUÇÃO                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ Signed URLs (URLs com credenciais criptografadas)               │
│                                                                 │
│ Servidor gera: URL + GoogleAccessId + Signature + Expires       │
│ Browser acessa: Credenciais já no URL                           │
│ Resultado: ✅ Funciona sempre (99%+)                            │
│                                                                 │
│ Implementação: 3 arquivos, ~125 linhas, 1-2 horas              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
