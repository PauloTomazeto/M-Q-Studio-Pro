┌──────────────────────────────────────────────────────────────────┐
│ CAMADA 1: PYTHON SERVICE (Novo)                                  │
├──────────────────────────────────────────────────────────────────┤
│ Localização: python/image_transformer.py                         │
│ Port: 127.0.0.1:5000                                            │
│                                                                  │
│ Funcionalidade:                                                  │
│ ✅ Processa imagens: redimensiona, normaliza, valida             │
│ ✅ Remove EXIF (privacidade)                                      │
│ ✅ Salva cópia em cache local (/cache/images/)                  │
│ ✅ Retorna base64 processado e pronto                            │
│ ✅ Fallback seguro (continue sem Python se offline)             │
│                                                                  │
│ Endpoints:                                                       │
│ - GET /health → Status do serviço                               │
│ - POST /process → Processa imagem base64                        │
│ - GET /info → Informações e cache stats                         │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ CAMADA 2: NODE.JS SERVER (Modificado)                            │
├──────────────────────────────────────────────────────────────────┤
│ Arquivo: server.ts (modificações em POST /api/storage/upload)   │
│                                                                  │
│ Fluxo novo:                                                      │
│ 1. Browser → Node.js /api/storage/upload (sem CORS)             │
│ 2. Node.js → Python (processa se disponível)                    │
│ 3. Node.js → Firebase (Admin SDK, server-to-server)            │
│ 4. Firebase → storage.googleapis.com URL pública                │
│ 5. Node.js ← URL sem CORS                                       │
│                                                                  │
│ Nova rota GET /api/cache/image/:imageId                         │
│ └─ Serve imagens locais do cache (mais rápido)                 │
│                                                                  │
│ Benefícios:                                                      │
│ ✅ Sem CORS (upload via localhost)                              │
│ ✅ Processamento de imagem robusto (Python + PIL)               │
│ ✅ Fallback seguro se Python offline                            │
│ ✅ Backup local de imagens processadas                          │
│ ✅ Mais rápido (serve cache antes de Firebase)                  │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ CAMADA 3: CLIENTE (Modificado)                                   │
├──────────────────────────────────────────────────────────────────┤
│ Arquivo: src/services/storageService.ts (uploadImage)           │
│                                                                  │
│ Mudança crítica:                                                 │
│ ANTES: uploadBytes() Client SDK (CORS)                          │
│ DEPOIS: axios.post('/api/storage/upload') via proxy             │
│                                                                  │
│ Fluxo novo em uploadImage():                                     │
│ 1. Converter File → base64 (FileReader)                         │
│ 2. POST /api/storage/upload com base64                          │
│ 3. Receber URL pública do servidor                              │
│ 4. Salvar URL em Firestore                                      │
│                                                                  │
│ Componentes que usam:                                            │
│ ✅ UploadStep.tsx → upload de imagem original                   │
│ ✅ GenerationStep.tsx → upload de imagem referência             │
│ ✅ Todos recebem URLs sem CORS                                  │
│                                                                  │
│ Garantias:                                                       │
│ ✅ Backward compatible (funções retornam mesma assinatura)     │
│ ✅ Zero mudanças em componentes React                           │
│ ✅ Fallback automático se Python offline                        │
