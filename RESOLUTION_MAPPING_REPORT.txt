1. PROBLEMA IDENTIFICADO
================================================================================

Localização: src/services/imageGenerationService.ts, linha ~67

Código Original (INCORRETO):
  const apiResolution = resolution === '4k' ? '4K' : (resolution === '2k' ? '2K' : '1K');

Problema:
  - Apenas 2 resoluções mapeadas: '4k' → '4K' e '2k' → '2K'
  - Resoluções '3k' e '2.5k' caíam no fallback '1K'
  - Usuário selecionava 3k/2.5k mas imagem gerava em 1K
  - Desperdício de créditos do usuário

================================================================================
2. SOLUÇÃO IMPLEMENTADA
================================================================================

2.1 NOVO RESOLUTION_MAP (imageGenerationService.ts, linhas 11-17)
────────────────────────────────────────────────────────────────

  const RESOLUTION_MAP: Record<string, string> = {
    '4k': '4K',
    '3k': '3K',
    '2.5k': '2.5K',
    '2k': '2K',
    '1k': '1K'
  };

Benefícios:
  ✓ Todas as 5 resoluções mapeadas explicitamente
  ✓ Fácil de manter e estender
  ✓ Centralizado em uma única estrutura
  ✓ Type-safe (Record<string, string>)

2.2 NOVA FUNÇÃO mapResolution() (imageGenerationService.ts, linhas 25-34)
──────────────────────────────────────────────────────────────────────────

  const mapResolution = (resolution: string): string => {
    const mapped = RESOLUTION_MAP[resolution.toLowerCase()];
    if (!mapped) {
      throw new Error(
        `Invalid resolution: ${resolution}. ` +
        `Supported resolutions: ${Object.keys(RESOLUTION_MAP).join(', ')}`
      );
    }
    return mapped;
  };

Benefícios:
  ✓ Validação de entrada com erro descritivo
  ✓ Case-insensitive (aceita '2K' ou '2k')
  ✓ Lança erro se resolução inválida em vez de fallback silencioso
  ✓ Mensagem de erro inclui resoluções suportadas

2.3 USO EM startGeneration() (imageGenerationService.ts, linha 97)
──────────────────────────────────────────────────────────────────

  Antes:  const apiResolution = resolution === '4k' ? '4K' : (resolution === '2k' ? '2K' : '1K');
  Depois: const apiResolution = mapResolution(resolution);

Benefícios:
  ✓ Código mais limpo e legível
  ✓ Lógica centralizada em uma função
  ✓ Fácil de testar independentemente
