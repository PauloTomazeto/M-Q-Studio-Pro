2. PROBLEMA: Por que Gemini consegue mas browser não?
================================================================================

Browser Request:
  GET https://firebasestorage.googleapis.com/v0/b/bucket/o/image.jpg?alt=media
  ❌ Sem credenciais
  ❌ Browser faz OPTIONS preflight
  ❌ Firebase Storage rejeita (CORS ou Permission Denied)

Gemini Request (via Proxy):
  POST http://localhost:3000/api/kie/gemini
  {
    "image_url": "https://firebasestorage.googleapis.com/v0/b/bucket/o/..."
  }
  ✅ Servidor backend recebe
  ✅ Servidor envia para KIE API (externo)
  ✅ KIE API acessa Firebase Storage (pode usar admin creds ou é público)

4. COMPARAÇÃO: PUBLIC URL vs SIGNED URL
================================================================================

PUBLIC URL (getDownloadURL):
  https://firebasestorage.googleapis.com/v0/b/bucket/o/path%2Fto%2Ffile.jpg?alt=media

  Característica:      Status
  ─────────────────────────────────────
  Credenciais:         ❌ Nenhuma
  CORS Required:       ⚠️  Sim (depende config)
  Permissions Check:   ⚠️  Sim (regras storage)
  Expira:              ❌ Nunca
  Segurança:           ⚠️  Média (publicamente visível)
  Funciona no Browser: ❌ Falha se CORS/permission
  Funciona Gemini:     ✅ Funciona (via proxy admin)


SIGNED URL (getSignedUrl via Admin):
  https://firebasestorage.googleapis.com/v0/b/bucket/o/path%2Fto%2Ffile.jpg?
  GoogleAccessId=service-account@...&Signature=...&Expires=1234567890

  Característica:      Status
  ─────────────────────────────────────
  Credenciais:         ✅ Incorporadas
  CORS Required:       ❌ Não (creds authenticam)
  Permissions Check:   ❌ Não (creds já validadas)
  Expira:              ✅ Sim (default 7 dias)
  Segurança:           ✅ Alta (creds criptografadas, expirável)
  Funciona no Browser: ✅ Sempre funciona
  Funciona Gemini:     ✅ Sempre funciona


5. FLUXO DE TROUBLESHOOTING
================================================================================

Erro: "Response to preflight request doesn't pass access control check"
│
├─ Causa 1: CORS headers não retornados pelo Firebase
│  └─ Solução: Usar Signed URL (não precisa CORS)
│
├─ Causa 2: Storage Rules muito restritivos
│  └─ Solução: Usar Signed URL (credenciais incorporadas)
│
└─ Causa 3: Browser fazendo preflight (OPTIONS) para public URL sem creds
   └─ Solução: Usar Signed URL (credenciais fazem auth passar)


Erro: "Missing or insufficient permissions"
│
├─ Causa 1: Arquivo privado (default Firebase)
│  └─ Solução: Usar Signed URL (admin creds)
│
├─ Causa 2: User não autenticado no cliente
│  └─ Solução: Usar Signed URL (não precisa auth cliente)
│
└─ Causa 3: Storage Rules exigem userId match
   └─ Solução: Signed URL (gerado por admin, já validado)


Erro: "Max retry time for operation exceeded"
│
├─ Causa 1: URL expirada (muita retry)
│  └─ Solução: Regenerar Signed URL (7 dias válidade)
│
├─ Causa 2: Timeout durante download
│  └─ Solução: Usar proxy server para stream (servidor mais próximo)
│
└─ Causa 3: Network instável
   └─ Solução: Retry com exponential backoff
