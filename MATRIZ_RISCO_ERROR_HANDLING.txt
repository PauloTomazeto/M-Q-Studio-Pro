GAP 1: FALTA DE TIMEOUT NO POST /api/kie/nano-banana/create
───────────────────────────────────────────────────────────
Arquivo: server.ts (linhas 104-113)
Severidade: CRÍTICO
Probabilidade: OCASIONAL (3-5%)
Impacto: CRÍTICO - Usuário perde crédito sem resultado
Risco Total: MUITO ALTO

Descrição:
  - Requisição POST para criar tarefa NÃO tem timeout
  - Contrasta com Gemini que tem timeout de 120s
  - Se KIE API ficar lenta, requisição fica pendurada indefinidamente
  - Cliente espera resposta que nunca chega

Quando Ocorre:
  - Servidor KIE intermitente (timeouts em 3-5% dos casos)
  - Picos de carga na API KIE
  - Problema de rede entre MQ Studio e KIE API

Impacto em Produção:
  - Usuário vê spinner girando indefinidamente
  - 15-30 minutos depois, timeout do lado do cliente
  - Crédito foi consumido, nenhum resultado
  - Usuário irado, sem forma de diagnosticar

Custo de Não Fazer:
  - 1% das gerações fica pendurada
  - 10+ reclamações por dia em produção
  - Refund manual necessário

Custo de Fazer:
  - 2 minutos para adicionar timeout: 30000

RECOMENDAÇÃO: P0 IMEDIATO

GAP 3: POLLING INFINITO SEM MÁXIMO DE TENTATIVAS
──────────────────────────────────────────────
Arquivo: GenerationStep.tsx (linhas 184-237)
Severidade: CRÍTICO
Probabilidade: RARO (0.5-2%)
Impacto: CRÍTICO - Estado inconsistente indefinido
Risco Total: ALTO

Descrição:
  - setInterval para polling executa indefinidamente
  - Sem máximo de tentativas
  - Se status ficar 'processing' forever, polling nunca para
  - Sem limite de tempo global para geração

Quando Ocorre:
  - Tarefa fica presa em 'processing' na API KIE
  - Rate limit ou quota esgotada
  - Bug na API KIE

Impacto em Produção:
  - Gerador de imagem travado invisível para usuário
  - isGenerating = true para sempre
  - Memória vazada com interval não limpo
  - Usuário sem feedback
  - Só fecha/refresh para cancelar

Custo de Não Fazer:
  - 5-10 usuários por semana com "gerador travado"
  - Vazamento de memória se muito frequente
  - Difícil reproduzir e debugar

Custo de Fazer:
  - 10 minutos para adicionar MAX_POLLING_ATTEMPTS e timeout global

RECOMENDAÇÃO: P0 IMEDIATO

GAP 4: TRATAMENTO INCONSISTENTE DE response.data.code
─────────────────────────────────────────────────────
Arquivo: kieService.ts (linhas 613-627)
Severidade: ALTO
Probabilidade: OCASIONAL (2-4%)
Impacto: ALTO - Erro unclear para usuário
Risco Total: ALTO

Descrição:

  - Usuário não sabe qual é o erro real
  - Impossível diagnosticar

Custo de Não Fazer:

  - Mais tempo de suporte

Custo de Fazer:
  - 5 minutos para adicionar validação nula e melhor mensagem

GAP 7: REFUND INCORRETO - SEM DIFERENCIAÇÃO POR TIPO DE ERRO
────────────────────────────────────────────────────────────
Arquivo: GenerationStep.tsx (linhas 239-250)
Severidade: MÉDIO
Probabilidade: OCASIONAL (5-10% das falhas)
Impacto: MÉDIO - Usuário irado, falta confiança

GAP 8: HTTP 200 COM ERRO SEMÂNTICO
───────────────────────────────────
Arquivo: server.ts (linhas 116-120)
Severidade: BAIXO
Probabilidade: RARO (0.5%)
Impacto: BAIXO - Confusão em logs
Risco Total: BAIXO

Descrição:
  - KIE pode retornar HTTP 200 com code !== 200
  - server.ts passa direto sem validação

Impacto em Produção:

  - Client interpreta como sucesso temporariamente
  - kieService.ts logo após checa code !== 200 e lança erro

Custo de Não Fazer:
  - Confusão em debugging de logging
  - Raro mas possível

Custo de Fazer:
  - 5 minutos para validar code em server.ts

RECOMENDAÇÃO: P2 MÉDIO PRAZO (próximas 2 semanas)
