3. FLUXOS DE OPERAÇÃO
================================================================================

FLUXO 1: SUCESSO
  Crédito: -5
  Refund: Nenhum
  Resultado: ✅ Análise completa
  Saldo Final: -5

FLUXO 2: ERRO 5xx → RETRY SUCESSO
  Crédito: -5
  Refund: +5 (automático)
  Retry: Sem consumo adicional
  Resultado: ✅ Análise completa (após retry)
  Saldo Final: 0

FLUXO 3: TIMEOUT → RETRY FALHA
  Crédito: -5
  Refund 1: +5 (após erro)
  Refund 2: +5 (após retry falhar)
  Resultado: ❌ Erro final
  Saldo Final: +5 (usuário recebe bônus!)

FLUXO 4: REDO CONFIRMADO
  Modal: Confirmação requerida
  Crédito: -5 (após confirmar)
  Refund: Nenhum (se sucesso)
  Resultado: ✅ Nova análise
  Saldo Final: -5

FLUXO 5: REDO CANCELADO
  Modal: Confirmação mostrada
  Crédito: 0 (nenhum consumo)
  Resultado: Nenhum
  Saldo Final: 0

================================================================================
4. DETECÇÃO DE ERROS (isServerError)
================================================================================

O sistema detecta como erro de servidor:
  ✓ err.message.includes('5xx')
  ✓ err.message.includes('timeout')
  ✓ err.message.includes('HTML')
  ✓ err.response?.status >= 500

Esses erros acionam:
  1. Refund automático
  2. Retry automático (máx 1)
  3. Toast informativo

Erros que NÃO geram refund (usuário perde crédito):
  ✗ Imagem inválida
  ✗ Parâmetros ruins
  ✗ Créditos insuficientes
  ✗ Qualquer erro que não seja servidor

================================================================================
5. CONFIRMAÇÃO DE REDO
================================================================================

ANTES: handleRedo() → runAnalysis() direto
DEPOIS: handleRedo() → Modal → confirmRedo() → runAnalysis()

MODAL CARACTERÍSTICAS:
  - Ícone de aviso (warning color)
  - Título claro: "Refazer Análise"
  - Descrição: custo de 5 créditos
  - Aviso: "Esta ação não pode ser desfeita"
  - 2 botões: Cancelar | Sim, Refazer
  - Animação suave de entrada
  - Dark mode suportado
  - Responsivo em mobile

BOTÃO CANCELAR:
  → Fecha modal sem ação
  → Nenhum crédito consumido
  → Console: "[Diagnosis] User cancelled redo"

BOTÃO "Sim, Refazer":
  → confirmRedo() executa
  → Modal fecha
  → Toast: "Redo confirmado - 5 créditos serão deducted"
  → Resets: analysisStartedRef = false, retryCountRef = 0
  → Executa: runAnalysis()
  → Resultado: -5 créditos (novo diagnóstico)

8. PROTEÇÃO CONTRA DUPLA DEDUÇÃO
================================================================================

O sistema usa retryCountRef para garantir:

  ✓ Primeira tentativa (retryCount = 0)
    → consumeCredits(5) é CHAMADO

  ✓ Retry (retryCount = 1)
    → consumeCredits(5) é IGNORADO

  ✓ Máximo 1 retry (retryCount < 1)
    → Não há tentativa 3

RESULTADO: Máximo 5 créditos consumidos por diagnóstico

EXEMPLO:
  Erro → Refund → Retry OK
  Consumo: 1x (-5)
  Refund: 1x (+5)
  Saldo: 0 ✓
