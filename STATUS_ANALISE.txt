DESCOBERTAS PRINCIPAIS
================================================================================

PROBLEMA IDENTIFICADO:
  URLs públicas do Firebase Storage SEM credenciais causam CORS/Permission errors
  no browser, apesar de Gemini conseguir acessar via proxy do servidor.

RAIZ TÉCNICA:
  Firebase SDK cliente: getDownloadURL() retorna URL pública SEM creds
  ❌ Browser tenta GET → CORS preflight → Firebase rejeita
  ✅ Servidor via Admin SDK → Credenciais incorporadas → Funciona

SOLUÇÃO:
  Usar Signed URLs (URLs com credenciais criptografadas)
  ✅ Browser GET com creds → Firebase aceita
  ✅ Gemini API com creds → Funciona sempre
  ✅ Sem problemas CORS
  ✅ URLs expiram em 7 dias
