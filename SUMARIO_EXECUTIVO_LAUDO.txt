📊 VISÃO GERAL
═══════════════════════════════════════════════════════════════════════════════

Sistema está OPERACIONAL mas com FALHAS CRÍTICAS que causam:
  • Perda indefinida de créditos (5-10% das gerações)
  • Falhas silenciosas sem feedback ao usuário
  • Inconsistências com documentação da API

Impacto Estimado: ~$5-10/mês perdidos em créditos


═══════════════════════════════════════════════════════════════════════════════
🔴 3 PROBLEMAS CRÍTICOS
═══════════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────────┐
│ CRÍTICO #1: Polling Infinito (timeout sem limite)                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ Localização: src/components/studio/GenerationStep.tsx (linhas 184-237)     │
│                                                                             │
│ Problema:                                                                   │
│   • setInterval nunca para se status ficar 'processing'                    │
│   • 17.280 requisições GET/dia se task ficar processing                   │
│   • Créditos consumidos indefinidamente                                    │
│                                                                             │
│ Cenário de Falha:                                                           │
│   T=0h:  Usuário inicia geração → Status: 'processing'                    │
│   T=10h: Status ainda 'processing' → Polling ainda ativo                  │
│   T=24h: Status 'processing' → 17.280+ requisições feitas!                │
│                                                                             │
│ Fix Time: 5 minutos                                                         │
│           • Adicionar máximo de tentativas (ex: 120 = 10 minutos)          │
│           • Se timeout: falhar com erro claro                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ CRÍTICO #2: Validação Fraca de Parâmetros por Modelo                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ Localização: src/services/kieService.ts (linhas 595-618)                  │
│              src/components/studio/GenerationStep.tsx (linhas 32-40)       │
│                                                                             │
│ Problemas Encontrados:                                                      │
│                                                                             │
│ 1. Prompt Length Não Validado                                              │
│    • nano-banana-2: maxLength = 20.000 chars ❌ Sem validação             │
│    • nano-banana-pro: maxLength = 10.000 chars ❌ Sem validação           │
│    → Prompt de 12k chars em nano-pro → erro 422, créditos perdidos        │
│                                                                             │
│ 2. Aspect Ratio Inválido                                                   │
│    • nano-banana-2: suporta 15 tipos ['1:1', '1:4', '1:8', ...,'auto']  │
│    • nano-banana-pro: suporta 11 tipos ['1:1', '2:3', '3:2', ...]       │
│    → UI oferece lista única → usuário seleciona '4:1' → nano-pro falha   │
│    → erro 422, créditos perdidos                                          │
│                                                                             │
│ 3. Image Input Limit Não Validado                                          │
│    • nano-banana-2: maxItems = 14 imagens                                 │
│    • nano-banana-pro: maxItems = 8 imagens                                │
│    → Sem validação defensiva                                              │
│                                                                             │
│ 4. Output Format Hardcoded                                                  │
│    • Ambos retornam 'jpg' (deveria ser 'png' para pro)                   │
│    • Sem opção ao usuário                                                 │
│                                                                             │
│ Fix Time: 10 minutos                                                        │
│           • Validar prompt.length contra modelo                           │
│           • Validar aspect_ratio contra lista do modelo                   │
│           • Validar image_input.length contra modelo                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ CRÍTICO #3: Sem Timeout em Requisições HTTP                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ Localização: server.ts (linhas 95-138)                                     │
│                                                                             │
│ Problema:                                                                   │
│   • POST /api/kie/nano-banana/create: SEM timeout                        │
│   • GET /api/kie/nano-banana/status: SEM timeout                         │
│   • (Comparação: Gemini endpoint TEM timeout de 120s)                    │
│                                                                             │
│ Impacto:                                                                    │
│   • Se API KIE ficar lenta: requisição fica pendurada                     │
│   • Node.js acumula conexões abertas → Memory leak                        │
│   • Após 1-2 horas com muitas requisições: Servidor crasheia             │
│                                                                             │
│ Fix Time: 5 minutos                                                         │
│           • Adicionar axios timeout: 30000 em POST                        │
│           • Adicionar axios timeout: 10000 em GET                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘


═══════════════════════════════════════════════════════════════════════════════
🟠 PROBLEMAS ALTOS (Menos Severos)
═══════════════════════════════════════════════════════════════════════════════

⚠️ ALTO #4: Error Handling Ambíguo
   • Sem diferenciar erro 422 (client) vs 500 (server)
   • SEMPRE refunda créditos (mesmo para erros do usuário)
   • Log inadequado: "Polling error: [object Object]"
   • Fix: Diferenciar tipos de erro, refundar apenas servidor

⚠️ ALTO #5: Race Condition em Refund
   • Se clicar "Gerar" 2x: consumeCredits + consumeCredits
   • Se ambas falharem: refund + refund
   • Resultado: Créditos aumentam (bug)
   • Fix: Usar transação atômica no Firestore
