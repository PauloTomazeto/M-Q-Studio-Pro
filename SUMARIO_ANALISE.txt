RESUMO

8 GAPS IDENTIFICADOS no tratamento de erros do sistema de geração de imagens.

Severidade:
  ⚠️  CRÍTICOS: 3 (timeouts + polling infinito)
  🔴 ALTOS: 2 (validação de resposta)
  🟡 MÉDIOS: 2 (logging + refund)
  🟢 BAIXOS: 1 (HTTP 200 com erro)

Risco Geral: MUITO ALTO

Tempo de Fix: ~50 minutos
Tempo de Teste: ~30 minutos
Total: ~1.5 horas

================================================================================
OS 3 GAPS CRÍTICOS
================================================================================

1. FALTA DE TIMEOUT EM POST /api/kie/nano-banana/create
   Arquivo: server.ts (linhas 104-113)
   Problema: Requisição pode ficar pendurada indefinidamente
   Solução: Adicionar timeout: 30000
   Tempo: 2 minutos

2. FALTA DE TIMEOUT EM GET /api/kie/nano-banana/status/:taskId
   Arquivo: server.ts (linhas 124-138)
   Problema: Polling acumula requisições penduradas
   Solução: Adicionar timeout: 10000
   Tempo: 2 minutos

3. POLLING INFINITO SEM MÁXIMO DE TENTATIVAS
   Arquivo: GenerationStep.tsx (linhas 184-237)
   Problema: setInterval nunca para se status = 'processing'
   Solução: Adicionar MAX_POLLING_ATTEMPTS = 240 + timeout global
   Tempo: 10 minutos

================================================================================
OS 2 GAPS ALTOS
================================================================================

4. VALIDAÇÃO FRACA DE response.data.code/msg
   Arquivo: kieService.ts (linhas 613-627)
   Problema: Pode acessar properties de undefined
   Solução: Validar estrutura de resposta
   Tempo: 5 minutos

5. FALTA DE VALIDAÇÃO DE response.data.data
   Arquivo: kieService.ts (linhas 617, 627)
   Problema: TypeError "Cannot read property 'taskId' of null"
   Solução: Validar data !== null antes de acessar properties
   Tempo: 5 minutos

================================================================================
OS 2 GAPS MÉDIOS
================================================================================

6. LOG "Polling error: [object Object]" SEM CONTEXTO
   Arquivo: GenerationStep.tsx (linhas 234-236)
   Problema: Impossível debugar qual geração falhou
   Solução: Adicionar taskId, tentativa, status ao log
   Tempo: 5 minutos

7. REFUND SEMPRE (SEM DIFERENCIAR POR TIPO DE ERRO)
   Arquivo: GenerationStep.tsx (linhas 239-250)
   Problema: Refund mesmo em erro de validação/usuário
   Solução: Refund apenas em HTTP 5xx ou erro de rede
   Tempo: 10 minutos

================================================================================
O 1 GAP BAIXO
================================================================================

8. HTTP 200 COM CODE !== 200 (ERRO SEMÂNTICO)
   Arquivo: server.ts (linhas 116-120)
   Problema: Response HTTP 200 mas code !== 200
   Solução: Validar code em server.ts antes de retornar
   Tempo: 5 minutos
