Pergunta: O que foi implementado?
Resposta: Refund automático em erro servidor + Modal de confirmação para redo

Pergunta: Onde está o código?
Resposta: src/components/studio/DiagnosisStep.tsx (934 linhas, +105 mudanças)

Pergunta: Como funciona refund automático?
Resposta:
  1. Detecta erro 5xx/timeout/HTML
  2. Chama refundCredits(5)
  3. Mostra toast informativo
  4. Tenta retry automático

Pergunta: Como funciona redo com confirmação?
Resposta:
  1. Usuário clica "Refazer"
  2. Modal aparece: "Isto irá custar 5 créditos"
  3. Usuário escolhe: Sim ou Cancelar
  4. Se Sim: nova análise, -5 créditos

Pergunta: E se ambas tentativas falharem?
Resposta: Usuário recebe +5 créditos de bônus (refund duplo)

================================================================================
FLUXOS EM 1 LINHA
================================================================================

Sucesso:              -5 créditos

Timeout → Falha:      +5 créditos (bônus, refund duplo)
Redo OK:              -5 créditos adicionais
Redo Cancelado:       0 créditos

================================================================================
TROUBLESHOOTING
================================================================================

Problema: Refund não acontece
Solução: Verificar console [Credits] logs, verificar Firebase permissões

Problema: Modal não aparece
Solução: Verificar z-index, verificar showRedoConfirm state

Problema: Toast não aparece
Solução: Verificar Sonner está configurado, console errors

Problema: Créditos deducidos 2x
Solução: Verificar retryCountRef está resetando corretamente

Problema: Retry não tenta
Solução: Verificar isTransient, elapsedTime < 60, retryCountRef < 1
